# Wordle游戏预测仪表盘
## 功能描述文档

## 1. 概述

Wordle游戏预测仪表盘是一款基于Streamlit构建的交互式Web应用，用于可视化和分析各种Wordle游戏预测模型的结果。它提供了模型性能的全面视图、预测机制的详细洞察，以及模型选择和未来改进的可行建议。

## 2. 仪表盘架构

仪表盘采用模块化设计，各个部分清晰分离：

| 部分 | 目的 | 核心组件 |
|------|------|----------|
| **模型性能比较** | 比较不同模型在选定指标上的性能 | 交互式条形图、指标排序 |
| **模型详情** | 显示选定模型的详细指标 | 指标表格、比较表格 |
| **最佳模型推荐** | 按指标识别表现最佳的模型 | 最佳模型表格 |
| **模型特定可视化** | 显示单个模型的详细可视化 | 训练曲线、指标摘要、预测分析、误差分布 |
| **模型比较可视化** | 跨所有指标比较模型 | 全指标比较图、雷达图 |
| **模型洞察** | 提供架构和训练信息 | 模型架构表格、训练详情 |
| **注意力机制分析** | 可视化并解释注意力权重 | 注意力权重条形图、详细解释 |
| **结论和建议** | 总结发现并提供指导 | 关键发现、模型选择建议、研究优势、局限性、未来工作 |

## 3. 核心功能模块

### 3.1 模型性能比较

**目的**：可视化比较选定模型在不同指标上的性能。

**功能**：
- 交互式条形图，显示选定指标下的模型性能
- 按性能自动排序（误差指标升序，准确率指标降序）
- 颜色编码模型，便于识别
- 条形顶部显示数值
- 响应式设计，适应不同数量的选定模型

**支持的指标**：
- 平均绝对误差（MAE）
- 均方根误差（RMSE）
- 准确率
- F1分数
- 曲线下面积（AUC）

### 3.2 模型详情

**目的**：提供选定模型的详细性能指标。

**功能**：
- **单模型视图**：选定一个模型时，显示完整的指标表格
- **多模型视图**：选定多个模型时，显示比较表格
- 可根据选定指标排序的比较表格
- 格式化值，保留4位小数以确保精度

### 3.3 按指标的最佳模型

**目的**：快速识别每个指标的最佳模型。

**功能**：
- 自动确定每个指标的最佳模型
- 以清晰的表格格式显示结果
- 考虑误差指标越小越好，准确率指标越大越好

### 3.4 模型特定可视化

**目的**：为单个模型提供详细可视化，展示训练动态和性能特征。

**包含的可视化**：
- **训练曲线**：显示训练过程中损失和指标的变化趋势
- **指标摘要**：所有性能指标的综合视图
- **预测分析**：比较预测值与实际值，并显示误差分布
- **按活动水平的误差**：分析不同玩家活动水平的预测误差

**布局**：2x2网格布局，优化可视化展示

### 3.5 模型比较可视化

**目的**：提供所有模型在多个维度上的整体比较。

**包含的可视化**：
- **全指标比较**：比较所有模型在所有指标上的条形图
- **雷达图比较**：显示模型在所有指标上性能的多维雷达图

### 3.6 模型洞察

**目的**：提供架构和训练信息，以便更好地理解模型。

**组件**：
- **模型架构表格**：提供每个模型架构和关键特征的快速参考
- **训练信息**：关于批次大小、轮数、优化器和损失函数的详细信息

### 3.7 注意力机制分析

**目的**：可视化并解释注意力机制如何在预测时关注不同的序列步骤。

**功能**：
- **注意力权重条形图**：显示不同序列步骤的平均注意力权重
- **详细解释**：解释注意力权重的重要性，突出最关键的步骤
- **游戏机制上下文**：将注意力权重与实际Wordle游戏的认知过程联系起来

**关键洞察**：模型将最高重要性分配给中间尝试（第3步），这与Wordle游戏的"信息快速收敛阶段"一致。

### 3.8 结论和建议

**目的**：总结关键发现，并为模型选择和未来工作提供可行建议。

**组件**：
- **关键发现**：总结分析的主要洞察
- **模型选择建议**：为不同用例提供选择合适模型的指导
- **研究优势**：突出研究方法的优势
- **局限性和改进方向**：识别未来研究的领域
- **实用建议**：为不同应用场景提供指导

## 4. 交互功能

### 4.1 模型选择

- **多选功能**：用户可以选择一个或多个模型进行比较
- **默认选择**：默认选择所有模型，以便进行全面比较
- **实时更新**：模型选择更改时，仪表盘自动更新

### 4.2 指标选择

- **下拉菜单**：用户可以选择要可视化的指标
- **默认指标**：准确率，因为它通常是最直观的性能指标
- **指标名称**：以用户友好的格式显示（例如，"F1分数"而不是"f1_score"）

### 4.3 单个模型选择用于详细可视化

- **下拉菜单**：用户可以选择单个模型进行详细可视化
- **默认选择**：列表中的第一个模型
- **动态可视化加载**：图表根据选定的模型更新

### 5.1 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|
| Web框架 | Streamlit | 最新版 |
| 数据处理 | Pandas, NumPy | 最新版 |

## 6. 使用指南

### 6.1 启动仪表盘

```bash
# 在项目根目录下
python -m streamlit run wordle_lstm_project/dashboard.py
```

### 6.2 访问仪表盘

启动后，可通过以下地址访问仪表盘：
- **本地URL**：http://localhost:8502
- **网络URL**：可从同一网络的其他设备访问

### 6.3 使用仪表盘

1. **选择模型**：使用侧边栏选择一个或多个模型进行比较
2. **选择指标**：从下拉菜单中选择要可视化的指标
3. **查看模型详情**：查看模型详情部分，获取全面的指标
4. **探索可视化**：浏览各个可视化部分
5. **分析注意力权重**：查看注意力机制分析，获取可解释性洞察
6. **阅读建议**：查阅结论部分，获取关键发现和指导

## 7. 部署选项

### 7.1 本地部署

- **要求**：安装Python 3.10+、Streamlit和其他依赖
- **优势**：易于设置，完全控制数据和模型
- **使用场景**：开发、测试、内部团队使用

### 7.2 云部署

- **选项**：Streamlit Cloud、AWS EC2、Google Cloud Run、Microsoft Azure
- **优势**：可从任何地方访问，可扩展，无需本地安装
- **使用场景**：公开演示、协作研究、生产部署

## 8. 性能考虑

- **加载时间**：仪表盘加载迅速，具有优化的图像大小和高效的数据处理
- **响应式设计**：使用Streamlit内置的响应式布局，适应不同屏幕尺寸
- **交互更新**：实时更新，延迟最小
- **内存使用**：高效的内存管理，适用于大型数据集

## 9. 维护和更新

### 9.1 添加新模型

1. 确保新模型的评估结果在`model_evaluation_results.csv`中
2. 将模型特定的可视化文件放在适当的目录中
3. 如有需要，更新仪表盘代码中的模型颜色

### 9.2 更新可视化

1. 使用适当的脚本生成新的可视化文件
2. 替换相应输出目录中的旧文件
3. 在浏览器中刷新仪表盘，查看更新后的可视化

### 9.3 修改内容

- **文本内容**：直接在`dashboard.py`文件中更新
- **布局更改**：修改`dashboard.py`中的Streamlit组件结构
- **样式**：使用Streamlit的主题选项或自定义CSS进行定制

## 10. 结论

Wordle游戏预测仪表盘提供了一个全面、交互式的平台，用于分析和比较Wordle游戏预测模型。凭借其直观的界面、详细的可视化和可行的洞察，它成为研究人员、数据科学家和游戏开发者理解玩家行为和模型性能的宝贵工具。

仪表盘的模块化设计和响应式架构使其易于维护和扩展，允许随着研究的发展添加新的模型、指标和可视化类型。
